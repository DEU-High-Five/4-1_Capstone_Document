(POST) 아이디 중복확인


(POST)회원가입 시 전송될 데이터 (변수 이름) 명세
reg_id : 등록할 회원 아이디
reg_password: 암호화된 회원 비밀번호
reg_name: 사용자 이름
reg_phone: 사용자 전화번호. 10~11자리 숫자로만 구성된 문자열 (백엔드에서도 검증 필요, 시간 있으면 부탁)
reg_privacy_agreement : 개인정보 이용 동의 여부 (bool)
reg_consent_marketing : 마케팅 이용 동의 여부 (bool)

총 6개 데이터 입력

응답 데이터 명세
success : 정상적 회원가입 완료 혹은 입력 데이터 불량 여부 (bool)




(POST)로그인 시 전송될 데이터 (변수이름) 명세
userId: 로그인을 시도하는 사용자 ID
userPassword: 로그인을 시도하는 사용자 PW (암호화 안됨. 추후 SSL 인증서 등 고려)
==> 사용자 PW는 백엔드에서 Compare 해야 함

응답 데이터 명세
success : 로그인 성공 여부 (bool)
[로그인 성공 시 아래 user 전송 필요]
user = {
	Id : 로그인 성공한 사용자의 ID
	Name : 로그인 성공한 사용자의 이름
	accessToken : 발급된 액세스 토큰 (자동 로그인 용, RefreshToken은 서버에서만 관리)
}


토큰 관련 함수들(user.js)은 수정 없이 그대로 사용


(POST) 메인 홈 화면 접근 시 전송될 데이터 명세
userId : 사용자 아이디 (사용자 아이디로 본인 소속, 관리 기관 탐색)

응답 데이터 명세

myManagement: [		// 내 관리 기관 목록 (배열)
